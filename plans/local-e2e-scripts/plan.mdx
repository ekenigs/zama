# Local chain E2E scripts — fund, send, grant, verify

**Objective:** Add CLI scripts so a developer can run the full confidential-token loop on local Anvil + fhEVM: fund accounts with wrapped test tokens, submit `ConfidentialTransfer` transactions between arbitrary addresses, grant the indexer worker ACL delegation to decrypt third-party transfers, and verify outcomes via the existing read API and `decrypt_attempts` table.

**Done means:** With `pnpm dev` running (Anvil, deployed contracts, Postgres, indexer, worker, API), the six-step manual flow below completes without hand-written `cast` calls; `pnpm send --from … --to … --amount 1` accepts human decimal amounts; `pnpm fund` mints + wraps; `pnpm grant` registers on-chain delegation from a transfer party to the indexer EOA; the worker decrypts previously pending rows after grant and decrypts new third-party transfers on the first poll; `pnpm lint && pnpm typecheck && pnpm test && pnpm fallow` stay green.

**Non-goals:** Sepolia deployment, REST write endpoints, GitHub Actions / CI for this project, wallet UI, production KMS.

</RichText>

<RichText id="concrete-example">

### The six-step test you want to run

Assume Anvil account **#2** is Alice (`0x3C44…`), **#3** is Bob (`0x90F7…`), and the indexer worker uses account **#0** (`0xf39F…`, `INDEXER_PRIVATE_KEY` in `.env`).

```bash
pnpm dev:db && pnpm db:migrate && pnpm dev   # stack up

pnpm fund --address 0x3C44… --amount 100     # mint underlying + wrap → confidential balance
pnpm fund --address 0x90F7… --amount 10

# 1 — third-party transfer (indexer is neither party)
pnpm send --from 0x3C44… --to 0x90F7… --amount 1
# → indexer ingests row: amount_status=pending_decryption, amount_handle set

# 2 — worker fails (acl_denied logged in decrypt_attempts)
curl localhost:3000/v1/indexer/status          # pendingDecryptions ≥ 1
curl localhost:3000/v1/addresses/0x90F7…/transfers  # amount.status = pending_decryption

# 3 — Alice delegates decrypt rights to indexer
pnpm grant --from 0x3C44…

# 4 — worker backfill (same poll loop, no restart)
# → step-1 row becomes decrypted; decrypt_attempts shows success

# 5 — repeat transfer
pnpm send --from 0x3C44… --to 0x90F7… --amount 1

# 6 — new row decrypts on first worker attempt (attempt_num = 1, outcome = success)
```

`--wait` on `send` optionally blocks until API shows `decrypted` or a timeout.

<Callout id="decision-addresses" tone="decision">

**Test address map:** Scripts resolve Anvil dev keys for accounts #0–#9 by address (same keys as Foundry defaults). Indexer address is always derived from `INDEXER_PRIVATE_KEY` (Anvil #0) — never hard-coded and **never** a transfer party. All test transfers use **#2 → #3** (Alice → Bob); the indexer is always a third party.

</Callout>

<Callout id="decision-grant-model" tone="decision">

**Grant model (sender only):** `pnpm grant --from <sender>` requires `--from` to be the **transfer sender** (Alice). It calls ACL `delegateForUserDecryption(delegate=indexer, contract=CONTRACT_ADDRESS, expiration)` signed by that sender. The worker always uses **delegated decrypt** for `kind: transfer`, with `delegatorAddress = row.fromAddress`. Receiver-only grant is out of scope. Contract-scoped delegation covers future transfers from that sender without re-granting.

</Callout>

<Callout id="decision-fhevm-fetch" tone="decision">

**forge-fhevm auto-fetch (implemented in `dev.sh`):** First `pnpm dev` clones [forge-fhevm](https://github.com/zama-ai/forge-fhevm) into `vendor/forge-fhevm` (rev from `contracts/foundry.toml`), runs `forge soldeer install` in `contracts/`, then `./deploy-local.sh`. No manual `git clone`. Requires **git** and **network** on first run. `vendor/` is gitignored.

</Callout>

<Callout id="decision-funding" tone="decision">

**Funding (manual only):** `pnpm dev` does **not** auto-fund test accounts. The six-step flow includes explicit `pnpm fund` steps before sending. `dev.sh` only persists `UNDERLYING_ADDRESS` from deploy — no mint/wrap on startup.

</Callout>

<Callout id="decision-no-ci" tone="decision">

**No CI for this project:** Chain E2E is verified manually via the six-step script sequence and README. No GitHub Actions job, no `tests/e2e/chain-flow.test.ts`, no CI Foundry/fhEVM setup. Existing DB-level `tests/e2e/api.test.ts` stays for local `pnpm test`.

</Callout>

<RichText id="decrypt-worker-change">

### Worker change — delegation-first decrypt

The indexer is **never** a transfer party in the test model. Remove the party-only gate for transfers and use delegated decrypt with the sender as delegator. Wrap/burn keep direct `decryptValues` (zero-address parties). Before `pnpm grant`, transfers fail with `acl_denied`; after grant, the same poll loop backfills and decrypts new rows on the first attempt.

</RichText>

<RichText id="verification-detail">

### Verification

**Manual (primary):** Run the six-step script sequence above. Assert:

- After step 1: `SELECT outcome FROM decrypt_attempts WHERE transfer_id = …` shows `acl_denied` with `attempt_num >= 1`.
- After step 4: same `transfer_id` has `outcome = success`; API `amount.value` matches `1 * 10^6`.
- After step 6: newest transfer has exactly one `decrypt_attempts` row with `outcome = success` and `attempt_num = 1`.

**Automated:** Keep existing `tests/e2e/api.test.ts` (DB-seeded API contract only). No chain E2E test file — this project will not add CI.

**Quality gate:**

```bash
pnpm lint && pnpm typecheck && pnpm test && pnpm fallow
```

</RichText>

<RichText id="risks">

### Risks

- **forge-fhevm fetch fails:** If git clone or `deploy-local.sh` fails (no network, no git), `dev.sh` skips fhEVM deploy and wrap/send/decrypt will not work. Re-run `pnpm dev` after fixing connectivity; or remove `vendor/forge-fhevm` to force re-clone.
- **Indexer lag:** Allow 1–2 worker poll intervals (~4 s) between send and API assertions; `--wait` should use `WORKER_POLL_INTERVAL_MS * 3` default timeout.
- **Grant sender-only:** `pnpm grant --from` must be the transfer sender; worker never tries receiver as delegator.
- **Decimal vs wei:** Strictly parse `--amount` as decimal; never accept raw base units in `send`/`fund` to match your CLI spec.

</RichText>
